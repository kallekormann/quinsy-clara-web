What we process

When your monday.com automation runs, we receive and process the following data sent by monday:

  • Board ID (boardId)
  • Item ID (itemId)
  • Column ID (columnId)
  • Cell content (columnValue)
  • Selected format type (formatType)

We use short-lived monday API tokens provided during each automation run to read the current cell value and write back the formatted result.

What we do not do

  • We do not store cell content persistently.
  • We do not sell your data.
  • We do not use your data for advertising.
  • We do not build user profiles from board data.

Legal basis

Processing is based on the performance of the automation you configured (contract) and our legitimate interest in providing the formatting service you requested.

Sub-processors

  • Vercel (hosting): our API and website run on Vercel infrastructure in the EU/US depending on deployment region.
  • monday.com (platform): sends automation payloads and provides API access for read/write operations.

Retention

Cell data is processed in real time during each automation run and is not retained in our application database. Server logs on Vercel may temporarily contain request metadata. Log retention follows Vercel's default policies (typically up to 30 days for runtime logs).

Usage and billing

Clara tracks how many automation transformations you use through monday.com's app subscription API. We do not maintain a separate usage database or store cell content for metering. Usage limits depend on your monday marketplace plan. Payment and invoicing are handled by monday.com.

Your rights

Under GDPR, you may have the right to access, rectify, erase, restrict, or object to processing of your personal data, and to data portability where applicable. To exercise these rights, contact us at privacy@quinsy.app.

You may also lodge a complaint with your local data protection authority.

Changes

We may update this policy from time to time. The "Last updated" date at the top will reflect any changes.
